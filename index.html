// How It Works

Four phases.
One complete lifecycle.

01
// Phase_01 — Pre-Event
Build
// Construction

Student teams architect and construct JengaBank, a fictional fintech enterprise, under real constraints. Organic vulnerabilities emerge from genuine architectural decisions, not planted traps.

02
// Phase_02 — During Event
Maarifa
// Knowledge

Industry-led talks that directly shape what is attacked and defended. Morning sessions unlock attack vectors and defensive capabilities participants carry into the afternoon simulation.

03
// Phase_03 — During Event
Morani Hunting Grounds
// Mastery_Under_Pressure

Red teams execute full APT-style attacks on enterprise systems built by blue teams. Social engineering, cloud exploitation, insider threat mechanics. Depth of compromise measured, not binary.

04
// Phase_04 — During Event
Kikao
// Dialogue_&_Reflection

Industry leaders (CISOs, architects, technology executives) forensically review what actually happened. Real decisions, real attack chains. Followed by open Q&A on entering the industry.
